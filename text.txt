코드 -> 스테이징 -> 리포지토리


git add (파일명) : 스테이징하기
git commit -m "" : 커밋하기

git status : 스테이징한 파일 확인
git log --all --oneline : 커밋 기록들 확인

커밋할때 최근 커밋과 지금 수정한 파일의 차이점을 확인하고 하면 좋은데, 이걸 도와주는 기능들이 있음.
git diff : 터미널에서 보여주는데 가독성 별로라 안씀. 방향기 hjkl, q로 종료.
git difftool: 이걸 주로씀. 방향키 hjkl, :q 또는 :qa로 종료. 콜론 꼭 붙여야함! 

git difftool (커밋 아이디) : 해당 아이디에 속한 커밋과 현재 파일을 비교가능.
git difftool (커밋 아이디) (커밋 아이디): 커밋간 비교도 가능.

근데 그냥 difftool도 쓰지말고, vsc에 깔아둔 git graph 쓰셈.
